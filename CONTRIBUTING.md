# Contributing to GlycemicGPT Website

Thanks for your interest in contributing to the GlycemicGPT website. This guide covers how to set up your development environment, submit changes, and follow our quality standards.

## Development Setup

```bash
# Clone the repo
git clone https://github.com/GlycemicGPT/website.git
cd website

# Install dependencies
npm install

# Start dev server
npm run dev
```

The site runs at `http://localhost:3000` by default.

## Tech Stack

- **Next.js 16** with static export (`output: "export"`)
- **React 19**, TypeScript
- **Tailwind CSS v4**, shadcn/ui
- **Framer Motion** for animations
- **Recharts** for interactive charts
- Deployed to **GitHub Pages** via Cloudflare DNS

## Branching Model

All work follows a simple feature-branch workflow:

1. Create a feature branch from `main`: `git checkout -b feat/my-change`
2. Make your changes, commit with conventional commit messages
3. Push and open a PR targeting `main`
4. CI must pass before merge (lint, type check, build, Lighthouse)
5. Squash merge when approved

## Conventional Commits

All commit messages must follow the [Conventional Commits](https://www.conventionalcommits.org/) format:

```
type(scope): description

feat: add new hero animation
fix: correct chart tooltip positioning
docs: update installation guide
chore: bump dependencies
refactor: simplify architecture diagram component
perf: lazy load chart section
```

**Types:** `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `build`, `ci`, `chore`, `revert`

## No AI Attribution in Code

This project has a strict policy against AI tool attribution in commits, code, and PRs. This ensures all contributors take full ownership of their work.

**What's prohibited:**
- `Co-Authored-By` trailers referencing AI tools (Claude, Copilot, ChatGPT, etc.)
- Code comments like `// generated by AI`, `// copilot suggestion`, etc.
- PR descriptions containing AI tool branding (`Generated with Claude Code`, etc.)

**CI enforcement:**
- **CRITICAL** (auto-closes PR): Non-whitelisted bot co-author in commit trailers
- **HIGH** (blocks merge): AI tool name in commit trailers or PR description
- **MEDIUM** (blocks merge): AI attribution comments in code

**Whitelisted bots:** `github-actions[bot]`, `dependabot[bot]`, `coderabbitai[bot]`, `glycemicgpt-*[bot]`, `gitguardian[bot]`

## Code Style

- **TypeScript:** Strict mode, ESLint enforced
- **CSS:** Tailwind utility classes, use `cn()` helper for conditional classes
- **Components:** Functional components, named exports
- **Animations:** Respect `prefers-reduced-motion` via Framer Motion's `useReducedMotion`
- **Accessibility:** Semantic HTML, keyboard navigable, WCAG 2.1 AA

## CI/CD Pipeline

Every PR runs:
1. **Lint** (`eslint src/`)
2. **Type check** (`tsc --noEmit`)
3. **Build** (`next build` -- static export)
4. **Lighthouse** -- accessibility, best practices, SEO must score >= 90
5. **Attribution check** -- no AI tool attribution

Merges to `main` auto-deploy to GitHub Pages.

## PR Guidelines

- Keep PRs focused -- one feature or fix per PR
- Write a clear title using conventional commit format
- Describe what changed and why in the PR body
- Include screenshots for visual changes
- No AI attribution banners or tool branding in descriptions
